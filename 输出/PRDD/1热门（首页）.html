调用 大众点评API 获取饭约地点的经纬度信息，获取当前用户位置信息，计算显示饭约地点与用户地点的距离。

• 距离大于等于1km，显示为x千米，精确到小数点个位,若小数点为0不显示，四舍五入，例如 10.5km，3km

• 距离小于1km，大于100m，显示为X00m，精确到百位，四舍五入，例如300m

• 距离小于100m，显示为<100m

（DD/HH/mm）

1 <= DD < 2 显示为 明天结束

0 <= DD <1 显示为 今天结束

DD < -14 则该饭约不显示

3.22信息流排序规则（*未进行数字测试，初步模型）

信息流排序采用多维度计分排序，即根据以下规则计算获得每条饭约信息对应得分，按照得分多少进行降序排序。

 

score = 10*a + log5(b) + 5*log5(c) +10/d + 1/t2 + e（T1 < -14 * 24 无效）


• 信用值（a）★★★★★

      a = 信用值


• 截止时间（T1）★★★★★

      T1（hour） =  用户设置饭约时间 - 用户上一次刷新时间

 

 

• 发起时间（t2）★★★

      t2 （hour）= 当天的24点整时间 -  饭约最后一次编辑时间/发布时间

      t2最小为1

 

• 评论数（b）★★★

 

• 点赞数（c）★★★

 

• 饭约地点与用户距离（d）★

       d（km）= 饭约地点距离 - 用户当前位置

       d 最小为1


• 动态调节分值（e）★★★★★

       e为后台调节权重使用



排序结果基本上优先高信用值的饭约，其次是高评论数与点赞数，最后优先发帖时间与距离较近的。并且通过调节e值可改变整体排序得分

信息流首次载入时加载10条，用户浏览时记录用户最后浏览位置，短时间（以杀后台为时间点判断？）切换至其他页面再次切换回来（长时间相当于第一次加载页面则触发上拉刷新）
